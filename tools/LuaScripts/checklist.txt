emu: PARTIAL_SUPPORT_FINISHED / PLANNED
 "A library for interacting with the currently loaded emulator core"
 - currently lua calls _Update() once every frame rather then the script needing to call emu.frameadvance() to avoid freezing
 - functionaly emu.frameadvance() can be simulated by calling a co-routine from _Update and using yeild() instead.
 - not difficult to modify a script (usually only need to change one or two lines)
 - but since nearly all bizhawk scripts use emu.frameadvance() might be worth consideration

    Planned to Support:
     - frameadvance*

    No plans for support:
     - disassemble
     - displayvsync
     - frameadvance*
     - getboardname
     - getdisplaytype

    "These functions behavior identically to the mainmemory functions but the user can set the memory domain to read and write from.
     The default domain is the system bus. Use getcurrentmemorydomain(), and usememorydomain() to control which domain is used.
